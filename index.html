ScenEdit_SetWeather (temp, rate, percentage, state)
Set the current scene weather. This function takes 4 numbers that describe the desired weather conditions. These conditions are applied globally.

Parameters:

  • temp The current baseline temperature (in deg C). Varies by latitude.
  • rate The rainfall rate, 0-50.
  • percentage The amount of sky that is covered in cloud, 0.0-1.0
  • state The current sea state 0-9.

Usage:

    ScenEdit_SetWeather(math.random(0,25), math.random(0,50), math.random(0,10)/10.0, math.random(0,9))
ScenEdit_AssignUnitToMission (unit, mission)
Assign a unit to a mission. Takes a unit and mission descriptor, then assigns the unit to the mission if it exists. Fails silently if eitehr do not exist.

Parameters:

  • unit The unit to assign
  • mission The mission to execute

Usage:

    ScenEdit_AssignUnitToMission("Bar #1", "Strike")

ScenEdit_SetEMCON (type, name, emcon)

Sets the EMCON of the selected object. Select the object by specifying the type and the object's name.

Type is the type of object to set the EMCON on. It can be one of 4 values:

  • Side - Set an entire side's EMCON (e.g. United States using active radar)
  • Mission - Set the EMCON for a mission (e.g. Minesweepers active sonar)
  • Group - Set the EMCON for an entire group (e.g Package #20 active radar)
  • Unit - Set the EMCON for a single group (e.g. Hornet #14 passive radar)

emcon is a compound structure. The string follows the following grammar, with each clause separated by a semicolon (;)

  • Inherit indicates that the output EMCON should be at least the parent's EMCON. Inherit must come first.
  • A transmitter "set" statement. Each is of the form type=status, where type can be any one of Radar, Sonar, and OECM, and status can be any one of Passive or Active.

Parameters:

  • type The type of the thing to set the EMCON on.
  • name The object's name to select.
  • emcon The new EMCON for the object.

Usage:

  • ScenEdit_SetEMCON('Side', 'NATO', 'Radar=Active;Sonar=Passive')
  • ScenEdit_SetEMCON('Mission', 'ASW Patrol #1', 'Inherit;Sonar=Active')
  • ScenEdit_SetEMCON('Unit', 'Camel 2', 'OECM=Active')
ScenEdit_RunScript (script)
Run a script in a file. This function runs a script in the file script. This file must be inside the [Command base directory]/Lua directory, or else the game will not be able to load it.

Parameters:

  • script The script to run. The file that is looked for will be of the form [Command base directory]/Lua/[script]

Usage:

    ScenEdit_RunScript('cklib.lua')
ScenEdit_AddReferencePoint (descriptor)
Add a new reference point. Based on descriptor, a table of type ReferencePoint, this function adds a new reference point into the scene.

Parameters:

  • descriptor ReferencePoint The reference point to create.

Returns:

    ReferencePoint The reference point descriptor for the new reference point.

Usage:

    ScenEdit_AddReferencePoint({side="United States", name="Downed Pilot", lat=0.1, lon=4, highlighted=true})
ScenEdit_DeleteReferencePoint (selector)
Delete a reference point. Given a reference point selector, this function will remove it from the scene.

Parameters:

  • selector ReferencePoint The reference point selector describing the reference point to delete.
ScenEdit_SetReferencePoint (descriptor)
Update a reference point with new values. Given a descriptor with a valid selector, the function will find the correct reference point and update it to the values contained in the descriptor. Values may be omitted from the descriptor if they are intended to remain unmodified.

Parameters:

  • descriptor ReferencePoint A descriptor with a valid selector.

Usage:

  • ScenEdit_SetReferencePoint({side="United States", name="Downed Pilot", lat=0.5})
  • ScenEdit_SetReferencePoint({side="United States", name="Downed Pilot", lat=0.5, lon="N50.50.50", highlighted = true})

Tables

ReferencePoint
Reference point descriptor.

For creation, every field must be defined.

If selecting a unit (that is, using the descriptor as a selector) (e.g. SetReferencePoint or DeleteReferencePoint), one of the two following sets of fields must be define:

  • name and side, to select a reference point name belonging to side
  • guid, if the unique ID of the reference point is known.

If both are given, then the method defaults to finding the reference point with the correct GUID.

Fields:

  • name string The name of the reference point
  • latitude latlon The latitude of the reference point
  • longitude latlon The longitude of the reference point
  • guid string The unique identifier for the reference point
  • side string The side the reference point is visible to
  • highlighted bool True if the point should be selected
